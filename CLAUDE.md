# Portfolio — Claude Guidelines

## Voice and Tone

The site has one voice: **dry, witty, technically precise**. Think a senior engineer who is quietly funny — confident without performing confidence, self-aware without being self-indulgent.

- Short declarative sentences. No filler.
- Snark earns its place or it doesn't appear.
- Technical specificity is a feature, not a disclaimer.
- The writing page is intentionally warmer and more personal. That's acceptable — it's about writing, not systems.

## Snark Pool Rules (waybar-demo.html)

Each pool entry must carry a **distinct idea**, not a variation on the same idea.

- No exact duplicate strings across any pool entries, on any card.
- No semantic duplicates within the same pool (e.g., two entries that both say "cool temps" or "maybe hidden").
- No semantic duplicates across related cards (e.g., "AC power" and "charging" should not share lines or themes).
- Each card owns its metaphor space. The VPN card owns "disguise/hidden." Wired owns "reliability." Wi-Fi (no VPN) owns "exposure/surveillance." Don't bleed between them.
- When trimming, keep the sharpest and most distinct lines. Aim for 3–5 entries per pool; more than that invites bloat.

## Cross-Page Consistency

- **Email**: always `me@philipjrepko.com` — no gmail address.
- **Author name**: always "Philip Repko" in footers and meta.
- **Project names, job titles, and metrics** must match exactly across pages. If something appears on both the index and a detail page, both versions must be identical.
- Nav links: Work / Writing / Projects — consistent order and casing.

## What Not to Repeat

- Don't restate a concept in a heading and then immediately in the paragraph below it.
- Don't use the same punchline word as the emotional endpoint of two different cards (e.g., "wrong" closing both a CPU state and a weather state).
- Don't describe the page's purpose in a way that loops back on itself (e.g., "this projects page shows projects").

## Redundancy Check Before Adding Copy

Before adding a new snark line or descriptive phrase, ask:
1. Does this exact phrase or close variant already exist anywhere on this page?
2. Does this cover an idea already covered by another entry in the same pool?
3. Does this metaphor belong to a different card's established territory?

If yes to any: rewrite or cut.
